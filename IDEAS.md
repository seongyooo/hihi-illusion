# hihi 아이디어 노트

## 구현 완료

| 기능 | 설명 |
|------|------|
| **착시 (Illusion)** | 특정 카메라 각도에서 두 블록이 연결된 것처럼 보여 이동 가능 |
| **중력 반전 블록** | 밟으면 플레이어가 뒤집혀 블록 아랫면을 걷게 됨 |
| **맵 회전 블록** | 밟으면 맵 전체가 X 또는 Y축으로 회전 (각도 설정 가능) |
| **사다리** | 두 노드를 수직으로 연결 |
| **텔레포터** | 두 패드 사이 순간이동 |
| **스위치 (hold/toggle)** | hold: 서 있는 동안 활성 / toggle: 한 번 밟으면 on/off |
| **스위치 타입 (spawn/move)** | spawn: 블록 소멸/생성 / move: 블록을 지정 위치로 이동 |
| **엘리베이터** | 블록이 위아래로 자동 왕복 |
| **패트롤 블록** | 블록 자체가 일정 경로를 자동 이동 |
| **가시 블록** | always: 항상 위험 / blinking: 점멸하며 위험 구간 존재 |
| **적 — patrol** | 지정 노드 순서대로 순찰 |
| **적 — chase** | 일정 범위 내 플레이어를 BFS로 추적 |
| **적 — bounce** | X/Z축 방향으로 왕복, 착시로 연결된 엣지도 인식 |
| **레이저 블록** | 두 이미터 사이에 빔 생성, 접촉 시 사망, 스위치로 on/off |
| **챕터 선택 화면** | 30 스테이지씩 챕터로 묶어 선택 (Bounce Ball 스타일) |

## 미구현 아이디어

### 물 블록
- 물 안에 들어가면 이동 속도가 느려짐
- 또는 물에 빠지면 즉시 사망 (함정 타입)
- 시각적으로 반투명 파란 블록, 물결 애니메이션
- 착시와 조합: 물처럼 보이지만 실제론 건널 수 있는 환상

### 대포 블록
- 일정 간격으로 블록(포탄)을 한 방향으로 발사
- 포탄은 날아가다가 벽에 부딪히면 소멸
- 포탄 경로에 있으면 사망
- 타이밍 맞춰 지나가야 하는 메카닉

### 움직이는 레이저
- 이미터 자체가 patrol/elevator처럼 이동하는 레이저
- 회전하는 레이저 (한 이미터가 고정, 다른 이미터가 원형 궤도)
- 스윕 레이저: 일정 각도 범위를 왔다 갔다 스캔

### 거울 블록
- 레이저를 반사시키는 블록
- 배치 각도에 따라 반사 방향 결정
- 퍼즐: 레이저를 거울로 우회해서 특정 타깃을 맞춤

### 무게 추 / 시소
- 양쪽 끝이 연결된 블록 구조
- 한쪽을 밟으면 반대쪽이 올라가는 시소
- 적이나 포탄을 이용해 반대쪽 무게 조절

### 색깔 필터 블록
- 특정 색의 블록만 통과할 수 있는 게이트
- 스위치로 색 변경 → 통과 가능한 경로가 바뀜

### 얼음 블록
- 이동 시 관성이 생겨 멈추지 않고 미끄러짐
- 방향 계산이 필요한 퍼즐 요소

### 포탈 (고급 텔레포터)
- 텔레포트 후 이동 방향이 유지되거나 회전됨 (Portal 게임 스타일)
- 예: 위에서 들어가면 옆으로 나옴

### 시간 제한 블록
- 밟고 나서 N초 후 사라지는 블록
- 빠르게 건너야 하는 긴장감 요소

### 복사 캐릭터 (그림자 플레이)
- 과거 N초 전 내 이동 경로를 반복하는 잔상
- 잔상과 동시에 특정 위치를 점령해야 풀리는 퍼즐

---

## 맵 디자인 방향

- **챕터 1 (1~30)**: 기본 이동, 착시, 중력 반전 입문
- **챕터 2 (31~60)**: 레이저, 적, 맵 회전 조합
- **챕터 3 (61~90)**: 복합 메카닉 (레이저 + 착시 + 회전 등)
- 각 챕터 마지막 스테이지는 챕터 메카닉 총정리 보스 맵

---

## 메모

- 아이디어 우선순위: 움직이는 레이저 > 대포 블록 > 물 블록
- 새 메카닉 추가 시 에디터에도 동시 반영 필요
- 모바일 터치 입력 개선도 추후 고려
